Technical considerations in starting this blog

Places for writing that I considered were:

  • Jekyll
    • Found via this HN post (which also inspired me to start blogging)
    • I don’t have any experience with Ruby
    • At time of writing has 44k stars on github
  • Wordpress
    • Clunky and not customizable enough for me
    • My wife writes a wordpress blog in estonian about books ;)
  • Medium
    • Allows you to earn money by writing
    • Paywalls your posts ðŸ¤”
  • Hugo
    • Written in go (which i have experience in)
    • Fast (allegedly)
    • At time of writing has 56k stars on github

Comment section solutions considered were:

  • Disqus
    • Seemed too shady
    • Links with Github accounts
  • Cactus
    • Seemed cool
    • Allows anonymous comments out of the box

Hosting solutions considered were:

  • Freetier aws + nginx
    • Free
    • Probably handles traffic unless blog blows up
  • Some CDN (e.g aws Cloudfront)
    • Proper CDN solution
    • Good to get more experience with CDNs

Choices

So at the end of the day I chose to go with Hugo, since it’s golang based, had good tutorials and is more popular that jekyll. So far I’m very happy with the experience. Hugo also has lots of good themes to choose from.

I opted for Cactus since it has less tracking and allows anonymous comments out of the box.
But for some reason, the quick start didn’t work for me right out the gate.

And I decided to host this in a free-tier aws EC2 instance served by nginx, because a proper CDN solution seemed more complicated. Also I’m happy since I got more nginx experience. For example I hadn’t done proper Let’s Encrypt SSL before.
